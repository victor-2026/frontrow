# Accessibility Checklist

FrontRow takes accessibility seriously: test IDs ride alongside real screen-reader labels, they don't replace them.

## Per-component checklist

- [ ] `accessibilityLabel` describes the _purpose_ in human terms ("Buy ticket", not "buyButton")
- [ ] `accessibilityRole` matches the element's behaviour (`button`, `link`, `header`, `image`, …)
- [ ] `accessibilityHint` if the action is non-obvious ("Opens a payment sheet")
- [ ] `accessibilityState` reflects disabled / selected / busy state
- [ ] Touch target ≥ 44 × 44 pt (iOS HIG) / 48 × 48 dp (Android)
- [ ] Text scales with Dynamic Type (no fixed-size labels for content)
- [ ] Contrast ≥ 4.5 : 1 for body text, 3 : 1 for large text
- [ ] No content conveyed by colour alone

## Per-screen checklist

- [ ] Headings use `accessibilityRole="header"`
- [ ] Reading order matches visual order
- [ ] Modal dialogs trap focus and announce themselves
- [ ] Loading states are announced (`accessibilityLiveRegion`)
- [ ] All interactive elements are reachable by VoiceOver / TalkBack swipe

## Test surface

The current Debug Menu doesn't yet expose accessibility-specific toggles. Wishlist for a future iteration:

- Force large Dynamic Type
- Simulate VoiceOver / TalkBack focus order
- Inverted colours / high-contrast mode

In the meantime, exercise the real platform settings: iOS Simulator → Features → Accessibility Inspector, or Android emulator → Settings → Accessibility.
